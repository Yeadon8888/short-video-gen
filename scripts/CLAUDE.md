# scripts

## 目录结构

- `seed.ts`：初始化数据库种子数据，只处理本地开发起步。
- `deploy-supabase-cron.ts`：把 Supabase fallback cron 注册到数据库侧。
- `vercel-release.ts`：Vercel 发布前置校验与生产部署入口，负责检查链接状态并修正错误的 Root Directory。

## 架构决策

脚本目录只放两类东西：
开发起步脚本，与发布运维脚本。
发布脚本必须可重复执行、可诊断、可自修复，不能把关键知识藏在人的记忆里。

## 开发规范

- 新增部署脚本时，优先做“先检查再执行”，不要直接把危险动作串起来。
- 远端平台配置如果存在唯一正确值，可以在脚本里自动修正，并打印清晰日志。
- 脚本默认从仓库根目录运行，避免隐式依赖父目录。
