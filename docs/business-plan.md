# VidClaw — 商业计划书（Business Plan）

> 版本：2026-04-07 v1.0
> 定位：面向跨境与国内电商卖家的 AI 短视频批量生产平台

---

## 一、执行摘要

VidClaw 是一款 AI 驱动的短视频生成 SaaS 工具，专为电商卖家设计。核心价值是将「一张产品图 → 一条可直接发布的带货短视频（含脚本、文案、标签）」的全流程压缩到 1 分钟以内，单条成本低至 ¥1。

**已达成里程碑：**
- 产品已上线，核心生成流程完整闭环
- 已有 C 端付费用户（支付宝充值）
- 正在接入 Stripe 国际支付
- 已有代运营类 B 端客户在谈合作
- 支持 8 种输出语言，覆盖东南亚、拉美等跨境核心市场

**近期目标：**
- 完成 Stripe 接入，打开海外付费通道
- 签下首批 B 端代运营客户
- 12 个月内达到 MRR ¥100,000+

## 二、市场机会

### 2.1 市场规模

**短视频电商市场：**
- 2025 年全球短视频电商 GMV 突破 $500B（TikTok Shop + 抖音电商）
- 每个活跃店铺平均每月需要 30-100 条短视频内容
- 专业视频拍摄单条成本 ¥300-2,000，中小卖家难以承受

**目标可触达市场（SAM）：**

| 市场 | 活跃店铺数 | 月视频需求 | 潜在市场规模 |
|------|-----------|-----------|-------------|
| TikTok Shop（东南亚+欧美） | ~50 万 | 30-100 条/店 | $2B+/年 |
| 抖音电商（国内） | ~800 万 | 30-50 条/店 | ¥100B+/年 |
| 独立站+社媒运营 | ~100 万 | 20-50 条/店 | $500M+/年 |

**初期可服务市场（SOM）：**
聚焦 TikTok Shop 跨境卖家 + 国内抖音中小卖家，初期可触达约 10 万潜在用户，按 2% 转化率 + ¥50 ARPU 估算，初期年收入天花板约 ¥1,200 万。

### 2.2 用户痛点

1. **成本高**：专业拍摄一条带货视频 ¥300-2,000，中小卖家每月内容预算有限
2. **效率低**：手动做一条视频需要 2-4 小时（选题→脚本→拍摄→剪辑→发布）
3. **跨语言难**：跨境卖家需要多语言视频，翻译+配音成本极高
4. **无法批量**：50 个 SKU 的店铺，逐个制作视频根本做不完

### 2.3 竞争格局

| 竞品 | 定位 | 优势 | 弱点 |
|------|------|------|------|
| 剪映/CapCut | 通用视频编辑 | 用户基数大、免费 | 不生成内容，仍需人工创作 |
| Synthesia/HeyGen | AI 数字人视频 | 品牌知名度高 | 偏向企业宣传，非带货场景 |
| Creatify | AI 广告视频 | 广告素材生成 | 不支持批量、不支持多语言 |
| **VidClaw** | **电商带货视频** | **批量+多语言+完整文案** | **品牌知名度待建立** |

**核心差异化：** VidClaw 是唯一聚焦「电商带货」场景、支持「批量多商品 × 多语言」一键生成的工具。

## 三、产品能力

### 3.1 核心功能

**四种生成模式：**

| 模式 | 输入 | 适用场景 |
|------|------|---------|
| 主题原创 | 创意主题 + 产品图 | 新品上架、内容策划 |
| 链接二创 | 抖音/TikTok 爆款链接 + 产品图 | 跟爆款、蹭热点 |
| 视频上传二创 | 本地视频 + 产品图 | 素材改编、竞品参考 |
| 批量带货 | 批量主题 + N 张产品图 | 多 SKU 店铺批量铺量 |

**完整输出物（每条视频）：**
- AI 生成的短视频（4-15 秒，9:16 竖屏或 16:9 横屏）
- 分镜脚本（含场景描述、镜头语言、Sora Prompt）
- 发布文案（标题 + 正文 + 首评 + 标签）
- 一键复制，直接粘贴到抖音/TikTok 发布页

**多语言支持（8 种）：**
英语、墨西哥西语、西班牙语、马来语、马来西亚英语、巴西葡语、印尼语、阿拉伯语

### 3.2 技术架构

```
用户上传产品图
     ↓
AI 脚本生成（Gemini）→ 分镜 + Prompt + 文案
     ↓
视频生成（BLTCY/Plato · Yunwu 双供应商）
     ↓
智能补齐（失败自动重试，直到达标或超时）
     ↓
结果交付（在线预览 + ZIP 下载）
```

**关键技术优势：**
- **多供应商冗余**：一家限流自动切另一家，保障交付稳定性
- **智能失败分类**：内容审核失败（不重试）vs 供应商故障（自动重试）vs 超时（延迟重试）
- **批量节流引擎**：窗口化提交 + 2 秒间隔，防止大批量冲垮供应商限额
- **积分事务安全**：充值、消费、退款全链路事务一致性

### 3.3 管理后台

- 用户管理：搜索、角色切换、积分充值/扣减、封禁
- 模型管理：添加/编辑视频模型、设置积分单价、配置默认参数
- 支付配置：支付宝参数、套餐管理、价格调整
- 任务监控：全局任务状态、失败率、错误追踪
- 公告系统：面向全站用户发布通知

## 五、用户增长策略

### 5.1 获客渠道

**第一阶段：内容驱动获客（CAC ≈ ¥0）**

核心策略：**用自己的产品做自己的营销内容。**

- 每周发布 3-5 条「产品图 → 成品视频」对比视频
- 发布渠道：TikTok、YouTube Shorts、小红书、抖音
- 内容类型：
  - 「一张图变一条视频，只要 1 块钱」
  - 「帮我的客户批量生成了 50 条视频，全程 3 分钟」
  - 「跨境卖家怎么 0 成本做多语言视频？」

**第二阶段：社群+KOL（CAC ≈ ¥20-50）**

- 加入 TikTok Shop 卖家社群、跨境电商论坛
- 找 3-5 个电商类 KOL 体验分享
- 运营自己的卖家社群（微信群/Discord），提供免费额度换反馈

**第三阶段：渠道合作（CAC ≈ ¥0，分佣模式）**

- 与跨境电商培训机构合作（他们的学员就是我们的用户）
- 与代运营公司合作（B 端合作，详见 5.2）
- 代理商返佣 20-30%

### 5.2 B 端合作模型

代运营公司是最快的规模化路径——一个 B 端 = 几十个 C 端的消费量。

**合作方式：**

```
代运营公司（管理 50-500 个店铺）
     ↓
月度预存 / 月结合同
     ↓
通过 VidClaw 批量生成视频
     ↓
分发到各店铺的抖音/TikTok 账号
```

**B 端需求优先级（已识别）：**
1. 大额月结 / 预存（不按小额积分包）
2. API 接入（对接他们自己的工单系统）
3. 子账号体系（一个主账号管理多店铺）
4. 品牌白标（可选，高级合作模式）

**签约路径：**
- 第一个 B 端客户：给折扣价 + 手动服务，验证需求
- 标准化后：上线 B 端自助面板 + API
- 规模化后：渠道代理体系

### 5.3 用户转化漏斗

```
访问落地页
  ↓ （注册转化率 15-25%）
免费注册（邮箱 / Google 一键登录）
  ↓ （上传率 40-60%）
上传第一张产品图
  ↓ （试用率 60-80%）
首次生成体验（赠送积分 or 限时免费）
  ↓ （付费转化率 10-20%）
购买首充积分包（¥9.90 体验包）
  ↓ （复购率 30-50%）
持续充值 / 升级订阅
```

## 八、竞争壁垒与护城河

### 已建立的壁垒

1. **完整闭环**：市场上唯一同时输出「视频 + 脚本 + 文案 + 标签」的电商专用工具
2. **多语言能力**：8 种语言内置支持，跨境竞品几乎都缺失
3. **批量生产线**：N 商品 × M 条视频 一键批量，非简单 for 循环，而是有完整的节流/重试/补齐机制
4. **双供应商架构**：不被单一 AI 供应商绑定

### 需要建立的壁垒

1. **行业模板库**：按品类沉淀高转化创意模板（服装/3C/美妆/食品），让用户用了就离不开
2. **数据飞轮**：哪些 Prompt 出片效果好 → 优化默认参数 → 出片质量提升 → 更多用户 → 更多数据
3. **B 端锁定**：代运营客户一旦把 VidClaw 接入工作流，迁移成本极高
4. **品类know-how**：不同品类的视频风格差异很大，积累品类经验是竞争对手短时间难以复制的

## 九、风险分析与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| AI 视频供应商涨价/停服 | 高 | 高 | 多供应商架构已建立；监控成本，及时谈量价协议；长期考虑自建或接入开源模型 |
| 大厂推出同类产品 | 中 | 高 | 深耕电商垂直场景（批量+多语言+B端），做「卖家专用」而非通用工具；大厂通常做通用平台，垂直场景是盲区 |
| TikTok 被部分国家封禁 | 中 | 中 | 产品不绑定单一平台，已支持抖音+TikTok，可扩展到 YouTube Shorts、Instagram Reels |
| AI 生成内容被平台限流 | 中 | 中 | 持续跟踪平台政策；视频质量提升是关键——越接近真实拍摄效果，限流风险越低 |
| 用户付费意愿不足 | 中 | 高 | 降低首次付费门槛（¥9.9 体验包）；展示明确的 ROI（一条视频带来多少销量） |
| B 端客户需求过重 | 低 | 中 | 先用最轻方式服务（手动），验证后再产品化；不为单个客户做过多定制 |

## 十、团队与资源

**当前状态：** 独立创始人/小团队，全栈开发+运营

**技术栈优势：**
- 基于 Next.js 16 + Vercel 的现代架构，运维成本极低
- Supabase 全托管数据库+认证，无需专职 DBA
- 模块化设计，新功能开发效率高

**近期不需要额外人力的原因：**
- SaaS 模式边际成本低，100 用户和 1000 用户的运维成本差别不大
- AI 驱动，核心价值由算法而非人力交付
- 自动化程度高（自动充值、自动重试、自动退款）

**当 MRR > ¥50,000 时考虑：**
- 1 名客户成功/运营（处理 B 端关系 + 社群运营）
- 1 名兼职内容运营（生产获客内容）

## 十二、融资规划（可选）

### 当前阶段是否需要融资？

**短期内不需要。** 原因：
- 基础设施成本极低（< ¥1,500/月）
- 已有收入，接近或已达盈亏平衡
- 产品核心功能已完成，不存在"先烧钱做产品"的阶段

### 如果选择融资

**时机：** 当 MRR 稳定在 ¥50,000+ 且增长率 > 20% MoM 时，融资效率最高。

**用途：**
- 获客预算（内容投放 + KOL 合作）：60%
- 团队扩充（客户成功 + 运营）：30%
- 产品研发（B 端功能 + API）：10%

**估值参考：**
- SaaS 行业早期融资通常按 10-15x ARR
- 若 M12 ARR = ¥1,600,000，合理估值区间 ¥16M-24M（~$2.2M-3.3M）

### D. 关键指标定义

| 指标 | 定义 |
|------|------|
| MRR | 月经常性收入（Monthly Recurring Revenue） |
| ARPU | 每用户平均收入（Average Revenue Per User） |
| CAC | 客户获取成本（Customer Acquisition Cost） |
| LTV | 客户生命周期价值（Lifetime Value） |
| Churn | 月流失率（月初付费用户中本月未续费的比例） |
